Chapter 01

PyTorch vs TensorFlow — The Mental Model Shift

What is PyTorch?

PyTorch is an open-source deep learning framework by Meta AI. At its core it is a tensor computation library with automatic differentiation built in, plus a rich neural network API on top. Think of it as NumPy — but tensors can live on GPUs, and every operation remembers how to compute its own gradient.

Analogy
TensorFlow 1.x was like writing a recipe first and cooking later (define graph → run session). TF 2.x Eager Mode and PyTorch are like cooking live — every operation executes immediately. You can print(), use pdb, and add Python control flow freely.
TensorFlow 1.x (Static Graph)

Define the entire computation graph first. Execute via sess.run(). Fast at runtime, very painful to debug. You cannot inspect intermediate values without special ops.

PyTorch (Dynamic — Define-by-Run)

Graph built as operations execute. Every forward pass can have different shapes, branches, and Python control flow. Natural Python debugging with breakpoints and print statements.

Chapter 02

Tensors — The Atomic Unit of PyTorch

What is a Tensor?

A tensor is a generalization of arrays to any number of dimensions: a scalar is 0-D, a vector is 1-D, a matrix is 2-D, an image batch is 4-D ([N, C, H, W]). PyTorch tensors are like NumPy arrays but they can live on GPUs and track gradients.

Chapter 03

Autograd & Computational Graph — The Heart of PyTorch

What is Autograd?

Autograd (Automatic Differentiation) is PyTorch's engine for computing gradients. Every tensor operation builds a directed acyclic computation graph (DAG). When you call .backward(), PyTorch traverses this DAG in reverse using the chain rule to compute gradients for all leaf tensors with requires_grad=True.

Analogy
Imagine recording every turn and step on a hike. When you need to find the steepest descent (gradient), you replay the recording in reverse. Autograd records operations forward, then computes gradients by playing backward.
vs TensorFlow
TF2 requires with tf.GradientTape() as tape: to record operations. In PyTorch, recording is always on for tensors with requires_grad=True. No context manager needed for forward pass — just call .backward() at the end.
pythonAutograd Complete Walkthrough
import torch

# ─── requires_grad: opt-in to gradient tracking ──────────────────
# Leaf tensors you create: set requires_grad=True if they're parameters
x = torch.tensor(2.0, requires_grad=True)   # like a model weight
w = torch.tensor(1.0)                         # input data — no grad needed

# All downstream tensors inherit requires_grad=True automatically
y = x ** 2      # y.requires_grad = True; y.grad_fn = <PowBackward0>
z = y + 3      # z.grad_fn = <AddBackward0>

# grad_fn=None means leaf (created by user, not by an operation)
print(x.is_leaf, x.grad_fn)   # True, None
print(y.is_leaf, y.grad_fn)   # False, <PowBackward0>

# ─── Backward: compute gradients via chain rule ───────────────────
# dz/dx = dz/dy * dy/dx = 1 * 2x = 2*2 = 4
z.backward()         # z must be scalar, or pass gradient argument
print(x.grad)         # tensor(4.)  — stored on the leaf tensor

# ─── CRITICAL: gradients ACCUMULATE — zero them each step! ────────
x = torch.tensor(2.0, requires_grad=True)

(x ** 2).backward()
print(x.grad)   # tensor(4.)  correct

(x ** 2).backward()
print(x.grad)   # tensor(8.)  WRONG — accumulated!

x.grad.zero_()  # zero before next backward
(x ** 2).backward()
print(x.grad)   # tensor(4.)  correct

# ─── Non-scalar backward: pass a gradient tensor ─────────────────
x = torch.tensor([1., 2., 3.], requires_grad=True)
y = x ** 2   # vector, not scalar
y.backward(torch.ones_like(y))  # same as y.sum().backward()
print(x.grad)   # tensor([2., 4., 6.]) = 2x

# ─── Controlling gradient computation ────────────────────────────
# torch.no_grad(): disable tracking (for inference, metric computation)
with torch.no_grad():
    out = model(x)    # no graph built → faster, less memory

# torch.inference_mode(): stronger — also disables view tracking (PyTorch 1.9+)
with torch.inference_mode():
    out = model(x)   # preferred for inference

# .detach(): cut a tensor from the graph (shares data, no grad)
# Common use: plotting losses, target networks in RL
loss_val = loss.detach().cpu().numpy()   # safe to convert to numpy

# Freeze model layers (transfer learning)
for param in model.backbone.parameters():
    param.requires_grad = False

# ─── Higher-order gradients (meta-learning, Hessians) ────────────
x = torch.tensor(2.0, requires_grad=True)
y = x ** 3                                              # y = x³
dy_dx = torch.autograd.grad(y, x, create_graph=True)[0]  # 3x² = 12
d2y   = torch.autograd.grad(dy_dx, x)[0]               # 6x = 12
Computation Graph for z = x² + 3 when x = 2.0 x = 2.0 [leaf, requires_grad=True] | | forward: y = x² v y = 4.0 [grad_fn=PowBackward0] | | forward: z = y + 3 v z = 7.0 [grad_fn=AddBackward0] | | .backward() v Backward (chain rule): dz/dy = 1 (derivative of y+3 wrt y) dy/dx = 2x = 4.0 (derivative of x² wrt x) dz/dx = dz/dy * dy/dx = 1 * 4.0 = 4.0 → stored in x.grad

Chapter 3 Summary

  • Autograd builds a DAG dynamically during forward pass; .backward() traverses it in reverse
  • Gradients accumulate — always zero them before each backward (optimizer.zero_grad())
  • Use torch.no_grad() or torch.inference_mode() during evaluation — faster, less memory
  • .detach() cuts a tensor from the graph while keeping its data values
  • create_graph=True enables second-order gradients (meta-learning, MAML)
Chapter 04

Backpropagation — Manual & Automatic

Backprop Internals

Backpropagation is the chain rule of calculus applied to computational graphs. For a network with loss L and weights W, the gradient ∂L/∂W tells us how much each weight contributes to the error. We compute it by chaining together local gradients from the output back to each weight.

pythonManual Backprop — Understanding Every Step
import torch
import torch.nn.functional as F
torch.manual_seed(42)

N, D_in, H, D_out = 64, 10, 50, 1
x = torch.randn(N, D_in)
y = torch.randn(N, D_out)

# ─── MANUAL backprop (educational) ───────────────────────────────
W1 = torch.randn(D_in, H);   b1 = torch.zeros(H)
W2 = torch.randn(H, D_out);  b2 = torch.zeros(D_out)
lr = 1e-4

for t in range(5):
    # Forward pass
    z1     = x @ W1 + b1            # [N, H] linear
    h1     = z1.clamp(min=0)        # ReLU
    y_pred = h1 @ W2 + b2           # [N, 1]
    loss   = (y_pred - y).pow(2).mean()

    # Manual backward — chain rule by hand
    g_ypred = 2.0 * (y_pred - y) / N   # dL/dy_pred from MSE
    g_W2    = h1.T @ g_ypred          # dL/dW2
    g_b2    = g_ypred.sum(0)          # dL/db2
    g_h1    = g_ypred @ W2.T          # dL/dh1
    g_z1    = g_h1 * (z1 > 0).float() # dL/dz1 — ReLU grad: 1 if>0 else 0
    g_W1    = x.T @ g_z1              # dL/dW1
    g_b1    = g_z1.sum(0)             # dL/db1

    # SGD update
    W1 -= lr * g_W1;  b1 -= lr * g_b1
    W2 -= lr * g_W2;  b2 -= lr * g_b2
    print(f"Step {t}: {loss.item():.4f}")

# ─── AUTOMATIC backprop (what you always use) ─────────────────────
import torch.nn as nn

W1 = torch.randn(D_in, H,   requires_grad=True)
b1 = torch.zeros(H,         requires_grad=True)
W2 = torch.randn(H, D_out,  requires_grad=True)
b2 = torch.zeros(D_out,     requires_grad=True)

for t in range(5):
    y_pred = F.relu(x @ W1 + b1) @ W2 + b2
    loss   = (y_pred - y).pow(2).mean()

    # Zero accumulated gradients from previous step
    for p in [W1, b1, W2, b2]:
        if p.grad is not None: p.grad.zero_()

    loss.backward()   # PyTorch computes ALL gradients automatically

    with torch.no_grad():   # no_grad: update should not be tracked
        W1 -= lr * W1.grad;  b1 -= lr * b1.grad
        W2 -= lr * W2.grad;  b2 -= lr * b2.grad

Custom Autograd Function

pythonCustom backward — Straight-Through Estimator
class StraightThroughReLU(torch.autograd.Function):
    """ReLU forward, but gradient passes straight through.
    Used in: quantization-aware training, discrete relaxations."""
    @staticmethod
    def forward(ctx, x):
        ctx.save_for_backward(x)   # save for backward use
        return x.clamp(min=0)

    @staticmethod
    def backward(ctx, grad_output):
        # grad_output: gradient from downstream (dL/d_output)
        x, = ctx.saved_tensors
        return grad_output.clone()   # pass gradient unchanged

ste = StraightThroughReLU.apply
x   = torch.randn(5, requires_grad=True)
ste(x).sum().backward()
print(x.grad)   # all ones — gradient passes straight through

Chapter 4 Summary

  • Backprop = chain rule applied recursively backward through the computation graph
  • Core training pattern: forward → loss.backward() → optimizer.step() → zero_grad()
  • Always zero gradients before backward to prevent accumulation bugs
  • Custom Function subclasses let you define exotic backward passes (quantization, etc.)
Chapter 05

nn.Module — The Foundation of Every Model

What is nn.Module?

nn.Module is the base class for all neural networks in PyTorch — equivalent to keras.Model but with full transparency. Every layer, block, and full network is an nn.Module. Modules can be nested to build complex architectures.

pythonComplete nn.Module Anatomy
import torch.nn as nn
import torch.nn.functional as F

class MyNetwork(nn.Module):
    def __init__(self, input_dim, hidden_dim, output_dim):
        # ALWAYS call super().__init__() first
        super().__init__()

        # Layers assigned as attributes are auto-registered as submodules
        # Their parameters become part of model.parameters()
        self.fc1     = nn.Linear(input_dim, hidden_dim)   # W: [hidden, input] + bias
        self.bn1     = nn.BatchNorm1d(hidden_dim)
        self.dropout = nn.Dropout(p=0.3)
        self.fc2     = nn.Linear(hidden_dim, output_dim)

        # nn.Parameter: a tensor that is always a trainable parameter
        self.scale = nn.Parameter(torch.ones(1))

    def forward(self, x):
        # model(x) calls this via __call__, which also handles hooks
        x = self.fc1(x)          # linear transform
        x = self.bn1(x)          # batch normalization
        x = F.relu(x)           # activation (functional — no params)
        x = self.dropout(x)      # only active in training mode!
        x = self.fc2(x)
        return x * self.scale    # learned scaling

model = MyNetwork(784, 256, 10)
print(model)   # pretty-prints architecture tree

# Count parameters
total      = sum(p.numel() for p in model.parameters())
trainable  = sum(p.numel() for p in model.parameters() if p.requires_grad)
print(f"Total: {total:,} | Trainable: {trainable:,}")

# Access submodule parameters directly
print(model.fc1.weight.shape)   # [256, 784]
print(model.fc1.bias.shape)     # [256]

# Iterate over named parameters (used to build optimizer param groups)
for name, param in model.named_parameters():
    print(f"{name}: {param.shape}")

Sequential, ModuleList, ModuleDict

pythonContainer Modules
# nn.Sequential — layers applied in order (like keras.Sequential)
net = nn.Sequential(
    nn.Linear(784, 256), nn.ReLU(), nn.Dropout(0.2),
    nn.Linear(256, 128), nn.ReLU(),
    nn.Linear(128, 10)
)

# nn.ModuleList — for dynamic/indexed access to modules
# WRONG: self.layers = [nn.Linear(...)] ← plain list NOT tracked by PyTorch!
# RIGHT: self.layers = nn.ModuleList([nn.Linear(...)])
class DynamicNet(nn.Module):
    def __init__(self, n):
        super().__init__()
        self.layers = nn.ModuleList([nn.Linear(128, 128) for _ in range(n)])
    def forward(self, x):
        for layer in self.layers:
            x = F.relu(layer(x)) + x   # residual
        return x

# nn.ModuleDict — named sub-networks (multi-task, multi-head)
class MultiHeadModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.encoder = nn.Linear(128, 64)
        self.heads = nn.ModuleDict({
            'clf': nn.Linear(64, 10),
            'reg': nn.Linear(64, 1),
        })
    def forward(self, x, task):
        z = F.relu(self.encoder(x))
        return self.heads[task](z)

Weight Initialization

pythonInitialization — Critical for Stable Training
def init_weights(m):
    """Apply with: model.apply(init_weights)"""
    if isinstance(m, nn.Linear):
        nn.init.kaiming_normal_(m.weight, nonlinearity='relu')  # He init for ReLU
        if m.bias is not None: nn.init.zeros_(m.bias)
    elif isinstance(m, nn.Conv2d):
        nn.init.kaiming_normal_(m.weight, mode='fan_out', nonlinearity='relu')
    elif isinstance(m, (nn.BatchNorm2d, nn.LayerNorm)):
        nn.init.ones_(m.weight)    # gamma = 1
        nn.init.zeros_(m.bias)     # beta = 0

model.apply(init_weights)   # recursively applied to all submodules

# Quick reference:
# kaiming_normal_ / kaiming_uniform_  → ReLU, Leaky ReLU networks
# xavier_normal_  / xavier_uniform_   → tanh, sigmoid networks
# orthogonal_                         → RNNs
# trunc_normal_(std=0.02)             → ViT, modern Transformers

Chapter 5 Summary

  • Subclass nn.Module; always call super().__init__() first
  • forward() defines computation; model(x) calls it via __call__ (also handles hooks)
  • model.train() / model.eval() — switch modes! Dropout and BatchNorm depend on it
  • Use nn.ModuleList / nn.ModuleDict — plain Python lists are NOT tracked
  • Kaiming/He initialization for ReLU nets; Xavier for tanh/sigmoid

The Training Loop — Complete Production Pattern

The 5-Step Training Loop

Every PyTorch training loop follows the same 5 steps. Understanding each step deeply is what separates beginners from professionals.

The 5 Steps — Every Iteration: Step 1: Move data to device inputs.to(device), labels.to(device) Step 2: Zero gradients optimizer.zero_grad(set_to_none=True) Step 3: Forward pass outputs = model(inputs) loss = criterion(outputs, labels) Step 4: Backward + update loss.backward() clip_grad_norm_(model.parameters(), 1.0) optimizer.step() Step 5: Track metrics loss.item(), outputs.argmax(1)
pythonComplete Production Training Loop with AMP
import torch, torch.nn as nn, torch.optim as optim

def train_epoch(model, loader, optimizer, criterion, device, scaler):
    """Single training epoch. Returns (avg_loss, accuracy)."""
    model.train()   # CRITICAL: enables Dropout, updates BN running stats
    total_loss = 0.0; correct = 0; total = 0

    for inputs, labels in loader:
        # Step 1: Move to device
        inputs, labels = inputs.to(device), labels.to(device)

        # Step 2: Zero gradients (set_to_none=True is faster — frees memory)
        optimizer.zero_grad(set_to_none=True)

        # Step 3: Forward — wrapped in autocast for AMP (fp16 compute)
        with torch.autocast(device_type='cuda', dtype=torch.float16):
            outputs = model(inputs)
            loss    = criterion(outputs, labels)

        # Step 4: Backward — scaler handles fp16 gradient scaling
        scaler.scale(loss).backward()              # scaled backward
        scaler.unscale_(optimizer)                  # unscale before clipping
        nn.utils.clip_grad_norm_(model.parameters(), 1.0)
        scaler.step(optimizer)                      # update weights
        scaler.update()                             # update scaler state

        # Step 5: Track metrics (.item() extracts Python scalar from tensor)
        total_loss += loss.item()
        correct    += (outputs.argmax(1) == labels).sum().item()
        total      += labels.size(0)

    return total_loss / len(loader), 100.0 * correct / total


@torch.no_grad()
def evaluate(model, loader, criterion, device):
    """Validation loop — no gradients."""
    model.eval()   # CRITICAL: disables Dropout, uses BN running stats
    total_loss = 0.0; correct = 0; total = 0

    for inputs, labels in loader:
        inputs, labels = inputs.to(device), labels.to(device)
        outputs = model(inputs)
        loss    = criterion(outputs, labels)
        total_loss += loss.item()
        correct    += (outputs.argmax(1) == labels).sum().item()
        total      += labels.size(0)

    return total_loss / len(loader), 100.0 * correct / total


def train(model, train_loader, val_loader, epochs=50):
    device    = torch.device("cuda" if torch.cuda.is_available() else "cpu")
    model     = model.to(device)
    criterion = nn.CrossEntropyLoss()
    optimizer = optim.AdamW(model.parameters(), lr=3e-4, weight_decay=0.01)
    scheduler = optim.lr_scheduler.CosineAnnealingLR(optimizer, T_max=epochs)
    scaler    = torch.cuda.amp.GradScaler()  # AMP gradient scaler

    best_acc = 0.0
    patience_ctr = 0

    for epoch in range(1, epochs + 1):
        tr_loss, tr_acc = train_epoch(model, train_loader, optimizer, criterion, device, scaler)
        vl_loss, vl_acc = evaluate(model, val_loader, criterion, device)
        scheduler.step()

        print(f"Ep {epoch:3d} | tr {tr_loss:.4f}/{tr_acc:.1f}%"
              f" | val {vl_loss:.4f}/{vl_acc:.1f}%"
              f" | lr {optimizer.param_groups[0]['lr']:.2e}")

        # Checkpoint best model
        if vl_acc > best_acc:
            best_acc = vl_acc
            torch.save(model.state_dict(), 'best.pth')
            patience_ctr = 0
        else:
            patience_ctr += 1
            if patience_ctr >= 10:
                print("Early stopping."); break

    print(f"Best val accuracy: {best_acc:.2f}%")

CNN from Scratch — ResNet-Style Architecture

Building a ResNet Block

Residual connections are the single most important architectural innovation in deep learning since ReLU. They allow gradients to flow directly backward through skip connections, enabling networks to be hundreds of layers deep without vanishing gradients.

Analogy — Why Skip Connections Work
Imagine stacking 50 translators in a game of telephone. By the end, the original message is garbled. A residual connection is like having a hotline direct to the final translator — so even if the chain fails, the original signal still arrives intact.
pythonResidualBlock + Full ResNet for CIFAR-10
import torch, torch.nn as nn, torch.nn.functional as F

class ResidualBlock(nn.Module):
    """Basic ResNet block with identity shortcut.

    Architecture:
        Input → Conv3x3 → BN → ReLU → Conv3x3 → BN → (+Input) → ReLU
    The (+Input) is the residual / skip connection.
    If channels or spatial size differ, a 1x1 conv adjusts the shortcut.
    """
    def __init__(self, in_ch, out_ch, stride=1):
        super().__init__()
        # Main path: two 3x3 convs
        self.conv1 = nn.Conv2d(in_ch, out_ch, 3, stride=stride, padding=1, bias=False)
        self.bn1   = nn.BatchNorm2d(out_ch)
        self.conv2 = nn.Conv2d(out_ch, out_ch, 3, stride=1, padding=1, bias=False)
        self.bn2   = nn.BatchNorm2d(out_ch)

        # Shortcut: 1x1 conv to match dimensions when stride or channels differ
        self.shortcut = nn.Sequential()
        if stride != 1 or in_ch != out_ch:
            self.shortcut = nn.Sequential(
                nn.Conv2d(in_ch, out_ch, 1, stride=stride, bias=False),
                nn.BatchNorm2d(out_ch)
            )

    def forward(self, x):
        residual = x                              # save input
        out = F.relu(self.bn1(self.conv1(x)))    # conv → BN → ReLU
        out = self.bn2(self.conv2(out))           # conv → BN (no ReLU yet)
        out = out + self.shortcut(residual)       # ADD skip connection!
        out = F.relu(out)                         # ReLU after addition
        return out


class ResNet(nn.Module):
    """Mini-ResNet for CIFAR-10 (32x32 input, 10 classes).
    Architecture mirrors ResNet-18 at smaller scale.
    """
    def __init__(self, num_classes=10):
        super().__init__()
        # Stem: initial feature extraction (no downsampling yet)
        self.stem = nn.Sequential(
            nn.Conv2d(3, 64, 3, stride=1, padding=1, bias=False),
            nn.BatchNorm2d(64),
            nn.ReLU(inplace=True)
        )
        # Four stages: each doubles channels, halves spatial dims (except stage1)
        self.stage1 = self._make_stage(64,  64,  n_blocks=2, stride=1)
        self.stage2 = self._make_stage(64,  128, n_blocks=2, stride=2)
        self.stage3 = self._make_stage(128, 256, n_blocks=2, stride=2)
        self.stage4 = self._make_stage(256, 512, n_blocks=2, stride=2)
        # Classifier head
        self.pool = nn.AdaptiveAvgPool2d((1, 1))  # Global Average Pool → [N,512,1,1]
        self.fc   = nn.Linear(512, num_classes)

    def _make_stage(self, in_ch, out_ch, n_blocks, stride):
        # First block handles stride + channel change, rest keep same dims
        layers = [ResidualBlock(in_ch, out_ch, stride)]
        for _ in range(1, n_blocks):
            layers.append(ResidualBlock(out_ch, out_ch, stride=1))
        return nn.Sequential(*layers)

    def forward(self, x):
        x = self.stem(x)    # [N,3,32,32] → [N,64,32,32]
        x = self.stage1(x)  # [N,64,32,32]
        x = self.stage2(x)  # [N,128,16,16]
        x = self.stage3(x)  # [N,256,8,8]
        x = self.stage4(x)  # [N,512,4,4]
        x = self.pool(x)    # [N,512,1,1]
        x = x.flatten(1)   # [N,512]
        return self.fc(x)  # [N,10]

Understanding LSTM Internals

An LSTM has four gates per timestep: forget gate (what to drop from cell state), input gate (what new info to store), cell gate (candidate new values), and output gate (what to output). PyTorch's nn.LSTM implements all of this efficiently on GPU.

pythonSentiment Classifier with Bidirectional LSTM
import torch, torch.nn as nn, torch.nn.functional as F

class SentimentLSTM(nn.Module):
    """Text → Embedding → BiLSTM → Pool → Classifier.

    Uses pack_padded_sequence to handle variable-length sequences
    without wasting compute on padding tokens.
    """
    def __init__(self, vocab_size, embed_dim, hidden_dim,
                 n_layers, dropout, num_classes=2):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, embed_dim, padding_idx=0)
        self.lstm = nn.LSTM(
            input_size=embed_dim, hidden_size=hidden_dim,
            num_layers=n_layers, batch_first=True,
            dropout=dropout if n_layers > 1 else 0,
            bidirectional=True
        )
        self.dropout    = nn.Dropout(dropout)
        self.layer_norm = nn.LayerNorm(2 * hidden_dim)
        self.fc         = nn.Linear(2 * hidden_dim, num_classes)

    def forward(self, token_ids, lengths):
        # token_ids: [B, T] integer token IDs
        emb = self.dropout(self.embedding(token_ids))   # [B, T, E]

        # Pack: tells LSTM to skip padding tokens entirely
        packed = nn.utils.rnn.pack_padded_sequence(
            emb, lengths.cpu(), batch_first=True, enforce_sorted=False
        )
        packed_out, (hn, cn) = self.lstm(packed)

        # Unpack output (needed if we want all timestep outputs)
        out, _ = nn.utils.rnn.pad_packed_sequence(packed_out, batch_first=True)
        # out: [B, T, 2*H] — all timesteps

        # Option A: mean-pool over valid (non-padding) timesteps
        lengths_dev = lengths.to(out.device).float()
        pooled = out.sum(dim=1) / lengths_dev.unsqueeze(1)   # [B, 2H]

        # Option B: concatenate final forward + backward hidden states
        # hn shape: [num_layers*2, B, H] for bidirectional
        # hn[-2] = final forward; hn[-1] = final backward
        # pooled = torch.cat([hn[-2], hn[-1]], dim=1)  # [B, 2H]

        out = self.layer_norm(pooled)
        out = self.dropout(out)
        return self.fc(out)   # [B, num_classes]

class BahdanauAttention(nn.Module):
    """Additive / Bahdanau attention.
    Computes alignment scores between decoder hidden state and
    all encoder outputs, then creates a weighted context vector.
    """
    def __init__(self, hidden_dim):
        super().__init__()
        self.W_enc = nn.Linear(hidden_dim * 2, hidden_dim)   # encoder projection
        self.W_dec = nn.Linear(hidden_dim, hidden_dim)        # decoder projection
        self.v     = nn.Linear(hidden_dim, 1)                 # score vector

    def forward(self, decoder_hidden, encoder_outputs):
        # decoder_hidden:   [B, H]
        # encoder_outputs:  [B, T, 2H] (bidirectional)
        dec  = self.W_dec(decoder_hidden).unsqueeze(1)  # [B, 1, H]
        enc  = self.W_enc(encoder_outputs)              # [B, T, H]
        score = self.v(torch.tanh(dec + enc)).squeeze(2)  # [B, T]
        attn_weights = F.softmax(score, dim=1)          # [B, T] — sums to 1
        context = torch.bmm(attn_weights.unsqueeze(1), encoder_outputs)
        # context: [B, 1, 2H] — weighted sum of encoder states
        return context.squeeze(1), attn_weights

Multi-Head Self-Attention

The Transformer's core innovation is the ability to attend to any position in a sequence simultaneously. Multi-head attention runs several attention operations in parallel (each head learns different relationship patterns), then concatenates and projects their outputs.

pythonMulti-Head Attention + Transformer Encoder Block
import torch, torch.nn as nn, torch.nn.functional as F, math

class MultiHeadAttention(nn.Module):
    """Scaled dot-product multi-head attention.
    From: 'Attention is All You Need' (Vaswani et al., 2017)

    score(Q, K) = softmax(Q @ K.T / sqrt(d_k)) @ V
    Multiple heads learn different attention patterns in parallel.
    """
    def __init__(self, d_model, n_heads, dropout=0.1):
        super().__init__()
        assert d_model % n_heads == 0
        self.d_k    = d_model // n_heads
        self.n_h    = n_heads
        self.W_q    = nn.Linear(d_model, d_model, bias=False)
        self.W_k    = nn.Linear(d_model, d_model, bias=False)
        self.W_v    = nn.Linear(d_model, d_model, bias=False)
        self.W_o    = nn.Linear(d_model, d_model)
        self.drop   = nn.Dropout(dropout)

    def forward(self, Q, K, V, mask=None):
        # Q, K, V: [B, T, d_model]
        B = Q.size(0)

        # Project and split into heads: [B, T, d_model] → [B, n_heads, T, d_k]
        def split_heads(x):
            return x.view(B, -1, self.n_h, self.d_k).transpose(1, 2)

        Q = split_heads(self.W_q(Q))
        K = split_heads(self.W_k(K))
        V = split_heads(self.W_v(V))

        # Scaled dot-product attention
        scale  = math.sqrt(self.d_k)
        scores = torch.matmul(Q, K.transpose(-2, -1)) / scale  # [B, h, T, T]

        if mask is not None:
            # mask=0 means "ignore this position" → set score to -inf
            scores = scores.masked_fill(mask == 0, float('-inf'))

        attn = self.drop(F.softmax(scores, dim=-1))   # attention weights
        out  = torch.matmul(attn, V)                   # [B, h, T, d_k]

        # Merge heads: [B, h, T, d_k] → [B, T, d_model]
        out = out.transpose(1, 2).contiguous().view(B, -1, self.n_h * self.d_k)
        return self.W_o(out)   # final linear projection


class TransformerBlock(nn.Module):
    """One encoder block: Pre-LN → Self-Attn → residual → Pre-LN → FFN → residual.

    Pre-LN (normalize BEFORE attention) is more stable than
    Post-LN for deep models. Used in GPT-2, LLaMA, modern LLMs.
    """
    def __init__(self, d_model, n_heads, d_ff, dropout=0.1):
        super().__init__()
        self.attn  = MultiHeadAttention(d_model, n_heads, dropout)
        self.ffn   = nn.Sequential(
            nn.Linear(d_model, d_ff),
            nn.GELU(),                 # GELU — used in GPT, BERT (smoother than ReLU)
            nn.Dropout(dropout),
            nn.Linear(d_ff, d_model)
        )
        self.norm1 = nn.LayerNorm(d_model)
        self.norm2 = nn.LayerNorm(d_model)
        self.drop  = nn.Dropout(dropout)

    def forward(self, x, mask=None):
        # Pre-LN self-attention + residual
        x = x + self.drop(self.attn(self.norm1(x), self.norm1(x), self.norm1(x), mask))
        # Pre-LN FFN + residual
        x = x + self.drop(self.ffn(self.norm2(x)))
        return x


class TransformerClassifier(nn.Module):
    """Full Transformer encoder for sequence classification (like BERT)."""
    def __init__(self, vocab_size, d_model, n_heads, n_layers,
                 d_ff, max_seq_len, num_classes, dropout=0.1):
        super().__init__()
        self.embed    = nn.Embedding(vocab_size, d_model, padding_idx=0)
        self.pos_emb  = nn.Embedding(max_seq_len, d_model)   # learned positional
        self.drop     = nn.Dropout(dropout)
        self.blocks   = nn.ModuleList([
            TransformerBlock(d_model, n_heads, d_ff, dropout)
            for _ in range(n_layers)
        ])
        self.norm = nn.LayerNorm(d_model)
        self.fc   = nn.Linear(d_model, num_classes)

    def forward(self, x, mask=None):
        # x: [B, T] token ids
        B, T = x.shape
        pos  = torch.arange(T, device=x.device).unsqueeze(0)   # [1, T]
        x    = self.drop(self.embed(x) + self.pos_emb(pos))    # [B, T, d_model]
        for block in self.blocks: x = block(x, mask)
        x = self.norm(x)
        cls = x[:, 0, :]     # [CLS] token — or x.mean(dim=1)
        return self.fc(cls)

Chapter 14 Summary

  • Multi-head attention: project Q,K,V → split into n_heads → attend → merge → output project
  • Always scale attention scores by sqrt(d_k) — prevents softmax saturation
  • Pre-LN (normalize before attention) is more stable than Post-LN — use for deep models
  • GELU activation is standard in all modern Transformers (GPT, BERT, LLaMA)
  • Positional encodings (learned or sinusoidal) are essential — Transformers have no built-in order

import torch, torch.nn as nn, torch.optim as optim
from torchvision import datasets, transforms
from torch.utils.data import DataLoader
import numpy as np

# ─── Mixup augmentation ──────────────────────────────────────────
def mixup_data(x, y, alpha=0.4):
    """Mixes two training samples, creating virtual training examples.
    Regularizes model and improves calibration."""
    lam = np.random.beta(alpha, alpha) if alpha > 0 else 1
    idx = torch.randperm(x.size(0), device=x.device)
    mixed_x  = lam * x + (1 - lam) * x[idx]
    y_a, y_b = y, y[idx]
    return mixed_x, y_a, y_b, lam

def mixup_criterion(criterion, pred, y_a, y_b, lam):
    return lam * criterion(pred, y_a) + (1 - lam) * criterion(pred, y_b)

# ─── Warmup + cosine schedule ────────────────────────────────────
def get_lr_lambda(warmup_epochs, total_epochs):
    import math
    def fn(epoch):
        if epoch < warmup_epochs:
            return (epoch + 1) / warmup_epochs
        progress = (epoch - warmup_epochs) / (total_epochs - warmup_epochs)
        return 0.5 * (1 + math.cos(math.pi * progress))
    return fn

        optimizer.zero_grad(set_to_none=True)
        with torch.autocast('cuda', torch.float16):
            logits = model(x)
            loss   = mixup_criterion(criterion, logits, y_a, y_b, lam)
        scaler.scale(loss).backward()
        scaler.unscale_(optimizer)
        nn.utils.clip_grad_norm_(model.parameters(), 1.0)

        self.texts     = texts

        enc = self.tokenizer(
            self.texts[idx], max_length=self.max_len,
            padding='max_length', truncation=True, return_tensors='pt'
        )
        return {
            'input_ids':      enc['input_ids'].squeeze(),
            'attention_mask': enc['attention_mask'].squeeze(),
            'label':          torch.tensor(self.labels[idx], dtype=torch.long)
        }


class BERTClassifier(nn.Module):
    def __init__(self, model_name='bert-base-uncased', num_classes=2, dropout=0.1):
        super().__init__()
        self.bert    = AutoModel.from_pretrained(model_name)
        hidden_size  = self.bert.config.hidden_size   # 768 for bert-base
        self.dropout = nn.Dropout(dropout)
        self.head    = nn.Linear(hidden_size, num_classes)

        # Freeze all layers except last 2 transformer blocks + pooler
        for name, param in self.bert.named_parameters():
            if 'encoder.layer.10' not in name and 'encoder.layer.11' not in name and 'pooler' not in name:
                param.requires_grad = False

    def forward(self, input_ids, attention_mask):
        out     = self.bert(input_ids, attention_mask=attention_mask)
        cls_emb = out.last_hidden_state[:, 0, :]   # [CLS] token embedding
        return self.head(self.dropout(cls_emb))

class ModelEMA:
    """Maintains an exponential moving average of model weights."""
    def __init__(self, model, decay=0.9999):

        self.model = copy.deepcopy(model).eval()
        self.decay = decay

        with torch.no_grad():
            for ema_p, p in zip(self.model.parameters(), model.parameters()):
                ema_p.data.mul_(self.decay).add_(p.data, alpha=1 - self.decay)
